# 地火指路绘制逻辑（P5_YaoxingLuanwu.cs）

> 本文档梳理「地火安全点引导」整套绘制逻辑，**仅描述现有行为，不改变任何算法**。
> 行号引用 `Scripts/P5_YaoxingLuanwu.cs`（地火细分版，v0.0.20）。
> 配套阅读：底部「绘制出口与开关」一节说明四个细分开关如何挂接到这套逻辑上。

---

## 0. 名词与约定

| 记号 | 含义 |
| --- | --- |
| 场中 `ArenaCenter` | `(100, 0, 100)`，竞技场圆心 |
| 列 `col` | 地火读条按 X 坐标折算出的「斜向列号」，取值 1–7 |
| 候选列 `candidateCol` | 玩家选定策略对应的安全列（左半场 / 右半场各一个） |
| 相对列 `rel` | 当前这道火相对候选列的偏移，取值 0/1/2（mod 3） |
| `dir4` | 四方向枚举 0=N 1=W 2=S 3=E（见 `FireCardinalOffset`） |
| `dirN4` | 四斜角枚举 0=NW 1=SW 2=SE 3=NE（见 `FireDiagonalPoint`） |

核心思想：地火是 45° 斜向、逐波推进的火墙。玩家先按策略锁定一个**安全中心点**，
再根据每一波火「是否盖住安全列」推算出一条 3 段走位路线（起点 → 中转 → 安全点），
用「落点圈 + 连线 + 指路位移线」三层视觉呈现，并按命中时刻分时高亮。

## 1. 机制流程总览

```
StartCasting 47931  ─►  OnFireSafeGuideStart (行303)
                          └─ InitializeFireSafeGuide(reset=true)  画安全中心圈
StartCasting 47932  ─►  OnFireCast (行310)   ← 每道火一次，会多次触发
                          ├─ [EnableFireDraw]      画 steps 段步进危险圈
                          └─ [EnableFireSafeGuide] HandleFireSafeGuideCast(pos)
                                                    └─ 累积 rel，分派路线绘制
```

- `OnFireCast`（行310）每次自增 `_fireCount`，并解析 `SourcePosition` / `SourceRotation`。
- 危险圈：沿 `RotationToDirection(rot)` 方向，步距 `FireStepDistance=7`，半径 `FireCircleRadius=6`，
  画 `steps = Math.Clamp(FireStepCount, 1, 12)` 段；每段的 `DestoryAt` 按
  `castDuration + FireFirstExplosionDelayMs(700) + FireExplosionIntervalMs(700)*i` 推算，实现逐段消失。
- 安全引导：若未初始化则补 `InitializeFireSafeGuide(reset=false)`，再进 `HandleFireSafeGuideCast`。

## 2. 坐标系与候选列

### 2.1 策略 → 候选列：`FireCandidateColumns`（行634）

| 策略 | (左列, 右列) | 策略 | (左列, 右列) |
| --- | --- | --- | --- |
| Center | (3,3) | Left | (2,2) |
| Up | (4,2) | Right | (4,4) |
| Down | (2,4) | LeftUp | (3,1) |
| | | RightUp | (5,3) |
| | | LeftDown | (1,3) |
| | | RightDown | (3,5) |

`(0,0)` 表示该策略不启用引导（`_fireGuideReady=false`）。策略来源 `ResolveFireSafeStrategy`（行626）：
非 `Auto` 直接用设置值；`Auto` 时按自己在 `PartyList` 的位次查 `FireAutoStrategies`（MT/ST→Center，
H1→Left，H2→Right，D1/D2→Center，D3→Up，D4→Down）。

### 2.2 候选列 → 安全中心：`FireSafeCenter`（行651）

```
x = 70 + 5*leftCol + 5*rightCol
z = 100 - 5*leftCol + 5*rightCol
```

即以 `(70,100)` 为原点，`leftCol` 沿 (+5,−5)（东北向）、`rightCol` 沿 (+5,+5)（东南向）的
45° 斜基底。安全中心是「左半安全列」与「右半安全列」两条斜线的交点。
例：Center(3,3)→(100,100)；Up(4,2)→(100,90)；Left(2,2)→(90,100)。

### 2.3 几何偏移

- `FireCardinalOffset(c)`（行699）：0=N(0,−5) 1=W(−5,0) 2=S(0,+5) 3=E(+5,0)，幅度 `FireRouteOffset=5`。
  `FireCardinalPoint(c) = 安全中心 + offset`（四正向落点）。
- `FireDiagonalPoint(dirN4)`（行711）：0=NW 1=SW 2=SE 3=NE，分量 `FireDiagonalComponentOffset=2.5`
  （初始斜点 / 半程路线用的近距离斜向落点）。

## 3. rel 的产生与积累：`HandleFireSafeGuideCast`（行394）

每道火读条进来时：

1. `col = FireInitialColumn(pos.X)`（行656，`PositiveMod(Round((x-70)/5),7)+1`，1–7）。
   `if (col > 3) return;` — 只处理近侧列（1–3）的读条。
2. `isLeft = pos.X < 100`；据此取 `candidateCol = isLeft ? 左列 : 右列`。
3. **`rel = PositiveMod(col + 1 - candidateCol, 3)`**，存入 `_fireGuideCols` / `_fireGuideRels`，
   `n = 已累积条数`。

**rel 语义**：
- `rel == 0` → 这一波火**没盖住**安全列，原地即安全，无需移动；
- `rel != 0` → 安全列被盖，需要按路线挪到相邻安全位。

`rel` 序列（按时间先后）就是整套路线决策的输入。

## 4. 决策树：rel 组合 → 路线函数

> 守卫：`if (_fireRouteCompleted || n < 2) return;`（路线一旦定稿不再重画）。
> 另有「初始斜点」分支：`!_fireInitialPointDrawn && n<=3 && rel!=0 && 之前各波都 rel==0`
> → `DrawFireInitialPoint(n, rel)`（行522），先给一个早期落点提示。

| n | 条件（按 `_fireGuideRels`） | 调用 | 说明 |
| --- | --- | --- | --- |
| 2 | rel1≠0 且 rel2≠0 | `DrawFireFullRouteFrom12`（行455） | 头两波都要动 → 直接定整段路线 |
| 2 | rel1≠0 且 rel2==0 | `DrawFireFirstHalfRoute`（行482） | 先画前半，等后续补全 |
| 3 | rel1==0, rel2≠0, rel3≠0 | `DrawFireFullRouteFrom23`（行464） | 第1波安全，2/3 波定路线 |
| 4 | rel1==0, rel2==0, rel3≠0, rel4≠0 | `DrawFireFullRouteFrom34`（行473） | 前两波安全，3/4 波定路线 |
| 4 | `_fireFirstHalfDrawn` 且 rel1≠0, rel2==0, rel4≠0 | `DrawFireSecondHalfRoute`（行501） | 补全 n=2 留下的后半 |

### 4.1 整段路线三函数 `DrawFireFullRouteFrom12/23/34`

三者结构一致，区别在于用哪一对 rel 推参数：

```
clockwise   = (两 rel 相等)        // From23 取反：rel2 != rel3
dir4        = FireDir4FromRelPair(firstRel, secondRel)   // 行666
lateSafeCol = 候选列 + (后 rel==1 ? 1 : 0)
earlyDangerCol = 候选列 + (前 rel==1 ? 0 : 1)
DrawFireFullRoute(suffix, dir4, clockwise,
                  FireDelayForColumn(lateSafeCol) - FireWaveIntervalMs,   // delay1
                  FireDelayForColumn(earlyDangerCol))                     // delay2
```

`FireDir4FromRelPair`：(1,1)→3, (1,2)→0, (2,1)→2, (2,2)→1, 其他→0。

### 4.2 整段路线绘制 `DrawFireFullRoute`（行533）

1. `RemoveFireInitialPointDraws` 清掉早期斜点；钳制 `delay1≤delay2`，`total = max(FireGuideDurationMs, delay2+5000)`。
2. `route = FireFullRouteCardinals(dir4, clockwise)`（行678）取 3 个正向点序列：

   | dir4 | CCW(false) | CW(true) |
   | --- | --- | --- |
   | 0 | N→W→S [0,1,2] | N→E→S [0,3,2] |
   | 1 | W→S→E [1,2,3] | W→N→E [1,0,3] |
   | 2 | S→E→N [2,3,0] | S→W→N [2,1,0] |
   | 3 | E→N→W [3,0,1] | E→S→W [3,2,1] |

   即从某正向出发、绕半圈（顺/逆）到对向，经一个中间点。
3. 三点 `p1/p2/p3 = FireCardinalPoint(route[i])`；落点圈 ×3、连线 12 / 23 各一、指路位移线 ×3。

### 4.3 半程函数

- `DrawFireFirstHalfRoute(rel1)`（行482）：用 `FireDiagonalPoint(rel1==1?3:1)` 作起点 `from`，
  `to = 安全中心 + (安全中心 − from)`（对称到另一侧）；
  `delay = FireDelayForColumn(右列+1) - FireWaveIntervalMs`，置 `_fireFirstHalfDrawn=true`。
- `DrawFireSecondHalfRoute(rel1, rel4)`（行501）：`clockwise = rel1==rel4`，`dir4=FireDir4FromRelPair(rel1,rel4)`，
  取 `FireFullRouteCardinals` 的第 2、3 点续画后半，置 `_fireRouteCompleted=true`。

## 5. 时序模型

- `FireDelayForColumn(col) = FireHitBaseMs(4069) + FireHitIntervalMs(513) * col`（行661）：
  第 `col` 列被命中的时刻（自读条起算 ms）。列号越大、命中越晚，相邻列差 513ms。
- `FireWaveIntervalMs = 2500`：波与波间隔；路线 `delay1` 普遍减去一个波间隔，意为「在该波打到之前先就位」。
- 指路位移线分时高亮（以 `DrawFireFullRoute` 为例）：
  - `Guide1`：`Delay=0`，`DestoryAt=delay1` —— 先指向 p1（等待点）；
  - `Guide2`：`Delay=delay1`，`DestoryAt=delay2-delay1` —— 到时刻 delay1 改指 p2（中转）；
  - `Guide3`：`Delay=delay2`，`DestoryAt=total-delay2` —— 最后指 p3（安全点，用安全色）。
  形成「等待点 → 中转点 → 安全点」随时间推进的引导。

## 6. 绘制出口与开关（细分版新增）

所有落点 / 连线 / 指路最终只经由 **3 个原子绘制函数**，新增的细分开关即挂在这里：

| 原子函数（行） | 画什么 | 受控开关 |
| --- | --- | --- |
| `DrawFirePoint`（559） | 落点圆（安全中心 / 各路线点 / 初始斜点） | `EnableFireSafePoint` |
| `DrawFireRouteLine`（596） | 路线点之间的连线 | `EnableFireRouteLine` |
| `DrawFireGuideTo`（577） | 从自身指向目标的 Imgui 位移线 | `EnableFireGuideLine` |

另：
- `EnableFireSafeGuide`：安全引导**总开关**，关闭则上述整套不计算不绘制。
- `EnableFireDraw`：步进**危险圈**总开关，与安全引导相互独立。
- `FireStepCount`：危险圈段数（1–12，默认 7）；`AdvanceDrawMs`：危险圈提前量。

> 设计要点：三个子开关守卫只加在原子函数顶部（early-return），**完全不触碰**第 3–5 节的
> rel 累积与路线决策逻辑，因此默认全开时行为与原脚本逐像素一致。
