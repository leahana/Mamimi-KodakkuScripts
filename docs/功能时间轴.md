## 二、脚本实际覆盖的 4 大块（按首次出现排序）

1. **混沌洪水 / 地水**（0:26）— `49539`
   - `OnWater`：每道读条画矩形 AOE（宽 10×长 60，自 Boss 向后偏 30），延迟 4000ms。
   - `EnableFloodSafe`：每 4 道一组，按正/负旋转配对求两波交点作安全点（超 `FloodMaxDist` 丢弃），向场心偏 4m，带中转节点画「2→中→1」分时路线。

2. **癫狂交响曲 核爆 / 神圣**（0:39、2:11）— 状态 `5350` / `5351`
   - `OnBuster`（5350）：中标玩家身上核爆 r25 死刑圈，4000ms。
   - `OnSpread`（5351）：中标玩家身上神圣 r5 分散圈，5000ms。
   - `神圣引导预警`（v0.0.23 新增，挂在 `OnSpread`/`StatusRemove 5351`）：本批**首个** 5351 点名时，
     给其余未点名的 DPS+奶妈（队列索引 2~7）铺 r5 黄圈（可能的点名范围，防互撞/护炸），被点名者随各自
     5351 撤圈（还剩 3 人）；凑满 6 人（两轮 3+3）即清理，`StatusRemove` 逐人安全补删 + 超时 20s 兜底。
     开关 `EnableHolyGuide`（与分散圈解耦）/ `只画自己`（默认关）/ `Debug`。无「引导神圣」专属读条 ID，
     故以首个点名为预警起点（首个被点名者直接进真实分散圈、不画预警）。
   - ⚠️ 第三步「顺手加个核爆/神圣」（自身 25m/5m）是否复用同一状态待核实，见第四节。

3. **三星踩塔 + 钢铁 / 月环**（1:09–1:21，共 3 轮）
   - `47938` `OnTowerMechanicStart`：延迟后锁「闲人」（无易伤者）。
   - `ObjectEffect Id1:16` `OnTowerObjectEffect`：收集塔（火 2015294 / 冰 2015295 / 雷 2015296），满 4 座触发分配。
   - 易伤 `2902 / 2903 / 2998` `OnTowerDebuff`：记录玩家冰/火/雷易伤。
   - 分配：带新易伤者**顺时针**找第一座异色塔；闲人以 A 点为 12 点**逆时针**进双色塔；指路线 6500ms，可选头标。
   - 二选一灾祟：`49742` 钢铁（地震）实心圈 r10；`49743` 月环（龙卷）内圈 r10 安全 + 外圈 10–50 危险；8000ms，挂 Boss。

4. **混沌末世 地火步进 + 混沌涡旋**（2:02）— `47931` / `47932` + `47934`（本会话改造重点）
   - `47931` `OnFireSafeGuideStart`：按策略锁安全中心点并画圈。
   - `47932` `OnFireCast`（×6）：`EnableFireDraw` 画 `FireStepCount`（默认 7）段步进危险圈、逐段消失；安全引导按读条列号累积 `rel` 推算 3 段走位路线，分时亮箭头按命中时序推进。
   - `47934` `OnCastSpread` = 混沌涡旋：先 `RemoveFireSafeGuideDraws` 清地火，再给前 8 人各画 r5 分散圈。
   - **地火细分开关**：`EnableFireSafePoint`（落点圈）/ `EnableFireRouteLine`（走位连线）/ `EnableFireGuideLine`（指路箭头）/ `FireRouteDimStatic`（只亮当前段：隐藏连线 + 落点圈淡显）/ `FireRouteDimAlpha` / `FireStepCount` / `FireSafeDirection`。

## 三、脚本不处理的机制（无需绘制 / 他脚本负责）
- **连续究极**：4 连全屏 AOE，不可躲，靠减伤/治疗。
- **魔击**：TDH 分摊死刑平 A，按职能分组站位，无范围可画。
- **遗弃末世 / 遗弃末狱 / 遗弃末点**：狂暴阶段，由仓库另一脚本 `P2_EvenTowerRealtimeFans`（遗弃末世偶数塔扇形）负责。
